FUSION 是一个开箱即用的核物理科研智能体：懂每一个开源核物理程序，自带 62,714 篇 arXiv nucl-th 全文语料，跑在你能用的任何大模型上，包括国产模型。 你身后的每一个光点，都是语料库里一篇真实的论文。 FUSION is a research agent for nuclear physics that works out of the box: an expert skill for every open-source nuclear code, a self-contained corpus of 62,714 arXiv nucl-th papers, running on whatever LLM you can actually access. Every point of light behind this text is a real paper in the corpus.

是什么 · WhatWhat it is

一个平台，三层结构 One platform, three layers

FUSION 不是又一个聊天机器人。它是 opencode 的品牌分叉（MIT，每周 CI 自动重基上游）， 加上核物理专属的定制层，加上永远属于你自己的私人层。引擎与领域彻底解耦：上游进化，FUSION 跟着进化。 FUSION is not another chatbot. It is a rebrand fork of opencode (MIT, weekly CI rebase onto upstream), plus a nuclear-physics customization layer, plus a private layer that never leaves your machine. Engine and domain are fully decoupled: upstream evolves, FUSION evolves with it.

为什么 · WhyWhy FUSION

通用 agent 缺的，恰好是科研需要的 What generic agents lack is exactly what research needs

通用编程 agent 不认识 FRESCO 的 namelist，不知道 TALYS 退出码不可信，也搜不到 1990 年代的 nucl-th 预印本。FUSION 把这些领域知识做成平台的一等公民。 A generic coding agent has never seen a FRESCO namelist, doesn't know TALYS exits 0 on fatal errors, and can't search a nucl-th preprint from the 1990s. FUSION makes this domain knowledge a first-class citizen of the platform.

SKILLS

每个程序，一个专家技能One expert skill per code

反应（FRESCO TALYS CCFULL）、结构（GSM KSHELL）、R 矩阵、天体网络……每个技能教会 agent 安装、写输入、运行、解析该程序，并且必须复现一个发表过的基准值才能上线。连"TALYS 出错也返回 0"这种坑都写在技能里。 Reactions (FRESCO TALYS CCFULL), structure (GSM KSHELL), R-matrix, astro networks… each skill teaches the agent to install, write inputs for, run, and parse one code, and must reproduce a published benchmark before it ships. Even traps like "TALYS exits 0 on fatal errors" are baked in.

KNOWLEDGE

自带整个领域的文献The field's literature, built in

62,714 篇 arXiv nucl-th 论文（61,357 篇全文，1992–2026），SQLite FTS5 + BM25 离线检索；预生成 markdown 知识库：每篇论文一页（摘要 + 全文摘要 + 引用链接），每个 PhySH 概念一页。agent 用 grep 就能浏览，不需要任何外部 API。 62,714 arXiv nucl-th papers (61,357 fulltext, 1992–2026), searched offline via SQLite FTS5 + BM25; plus a pre-generated markdown wiki: one page per paper (abstract + digest + citation links), one page per PhySH concept. The agent browses it with plain grep. No external API required.

MODELS

模型无关，生来如此Provider-agnostic by design

DeepSeek、Qwen、GLM、Claude、GPT 都能驱动。质量门槛在国产模型上验收：连不上国外 API 的用户不是被妥协的对象，而是设计的出发点。 Runs on DeepSeek, Qwen, GLM, Claude, or GPT. The quality gate was passed on a domestic model: users who cannot reach Western APIs are not an afterthought; they are the starting point of the design.

PRIVACY

私人层永远是你的The private layer stays yours

你的精读文献 wiki、研究档案、集群密钥挂载在私人层，FUSION 只定义接口，从不打包分发任何人的私人数据。平台开源，知识私有。 Your read-literature wiki, research profile, and cluster credentials mount into the private layer. FUSION defines the interface and never ships anyone's personal data. The platform is open; your knowledge is private.

语料地图 · Corpus mapThe corpus map

55,850 篇论文的引用星图 55,850 papers, laid out by citation

知识库不只是一个数据库，它有形状。每篇论文的位置由它引用谁、被谁引用决定；地形是论文密度，地名是空间上真正聚集的 PhySH 主题。引用关系只做布局约束，一条边都不画：这是全语料可视化在这个量级上唯一不退化成毛线团的做法。 The knowledge base is not just a database; it has a shape. Each paper sits where its citations put it; terrain is paper density, place names are PhySH topics that genuinely cluster. Citations constrain the layout, but not a single edge is drawn: at this scale it is the only way a full-corpus visualization avoids collapsing into a hairball.

位置即语义Position is meaning 引用邻接 → SVD → t-SNE。同主题论文的空间离散度只有全局的 0.66，真正的主题家园低至 0.34。Citation adjacency → SVD → t-SNE. Same-topic dispersion is 0.66 of global; true topic homelands reach 0.34.
地名承担身份Names carry identity 只有真正聚集的主题才获得地名；黑白打印和色觉缺陷下依然可读，可以直接进论文。Only genuinely clustered topics earn a name; readable in grayscale and for color-blind readers, paper-figure ready.
